OWASP API Security Top 10 — Attack Surface 階層図

認証・認可不備リスクをインフラ層からフィールド層まで分類

Layer 1 / Infrastructure
🖥️
Attack Surface サーバーインスタンス
API4
Unrestricted Resource Consumption
リソース消費の無制限（DoS・コスト爆発）
API7
Server Side Request Forgery (SSRF)
サーバーを踏み台に内部ネットワークへ到達
API8
Security Misconfiguration
デフォルト認証情報・不要ポート開放
API9
Improper Inventory Management
古いバージョン・放置エンドポイントが稼働
Layer 2 / Application
⚙️
Attack Surface アプリケーション（エンドポイント・機能）
API2
Broken Authentication
認証の不備 — トークン漏洩・セッション固定
API5
Broken Function Level Authorization
機能レベルの認可不備 — 管理者APIを一般ユーザーが呼び出し
API6
Unrestricted Access to Sensitive Business Flows
ビジネスフローの悪用 — ボットによる大量操作
API10
Unsafe Consumption of APIs
外部APIを無検証で信頼 — サプライチェーンリスク
Layer 3 / Data Model
🗂️
Attack Surface データモデル（オブジェクト・リソース単位）
API1
Broken Object Level Authorization (BOLA)
オブジェクト単位の認可不備 — 他ユーザーのリソースIDに差し替えてアクセス
Layer 4 / Field
🔬
Attack Surface フィールド（プロパティ・属性単位）
API3
Broken Object Property Level Authorization (BOPLA)
過剰なフィールド露出（旧：Excessive Data Exposure）＋ 不正な属性書き換え（旧：Mass Assignment）
全10項目 対応表
API1BOLA → Layer 3（モデル）
API2Broken Auth → Layer 2（アプリ）
API3BOPLA → Layer 4（フィールド）
API4Resource Consumption → Layer 1（サーバー）
API5BFLA → Layer 2（アプリ）
API6Business Flows → Layer 2（アプリ）
API7SSRF → Layer 1（サーバー）
API8Misconfiguration → Layer 1（サーバー）
API9Inventory → Layer 1（サーバー）
API10Unsafe API → Layer 2（アプリ）
