▸ ベクターの各項目を解説する

AV — Attack Vector(攻撃元)

NNetwork: インターネット越しなど、リモートから攻撃可能
AAdjacent: 同一セグメント/隣接ネットワークからのみ攻撃可能
LLocal: ローカルログインやシェルアクセスが必要
PPhysical: 機器への物理的な接触が必要

AC — Attack Complexity(攻撃条件の複雑さ)

LLow: 特別な条件なしに再現・成功させやすい
HHigh: 競合状態の悪用や事前偵察など特殊な条件が必要

PR — Privileges Required(必要な権限)

NNone: 権限不要で攻撃可能
LLow: 一般ユーザー相当の権限が必要
HHigh: 管理者/root相当の権限が必要

UI — User Interaction(ユーザー操作)

NNone: 被害者の操作なしに成立する
RRequired: リンクを開くなど被害者側の操作が必要

S — Scope(影響範囲)

UUnchanged: 影響が脆弱なコンポーネント内に留まる
CChanged: 権限境界を越え、他のコンポーネントにも影響が及ぶ

C / I / A — Confidentiality / Integrity / Availability(3要素への影響)

NNone: 影響なし
LLow: 一部の情報/機能に限定的な影響
HHigh: 全面的・重大な影響(全データ窃取、全停止など)

スコアの仕組み

式Base Score は「Exploitability(攻撃のしやすさ = AV・AC・PR・UI から算出)」と「Impact(実害の大きさ = C・I・A から算出)」を組み合わせて決まる。C/I/A が全て None だと Impact が 0 になり、他がどれだけ攻撃しやすくてもスコアは 0(なし)になる。
段階0 = なし / 0.1–3.9 = 低 / 4.0–6.9 = 中 / 7.0–8.9 = 高 / 9.0–10.0 = 緊急(Critical)
