01概要

X（旧Twitter）が無償・認証不要で提供する oEmbed エンドポイントを使い、ツイートの本文・投稿者情報を取得する。APIキー不要で呼び出せる一方、公式レート制限が非公開のため、エラー時の指数関数的リトライを実装して安定稼働させることが目的。

項目内容
エンドポイントhttps://publish.twitter.com/oembed
認証不要（APIキー不要）
公式レート制限非公開（実績値: 180 req / 15 min の報告あり）
取得できるデータ本文、投稿者名、投稿日時、埋め込みHTML
取得できないデータいいね数、RT数、メディアURL直リンク、鍵垢ツイート

02エンドポイント仕様

リクエスト

GET https://publish.twitter.com/oembed

パラメータ必須説明
url 必須 ツイートURL（URLエンコード必須）
lang 任意 表示言語（例: ja）。本文の言語は変わらない
omit_script 任意 true にすると widgets.js タグを除外。サーバー取得時は推奨
theme 任意 dark / light（埋め込みUIの見た目のみに影響）

03指数関数的リトライ設計

待機時間は以下の式で計算する。ジッター（乱数）を加えることで、複数プロセスが同時リトライしても集中しない。

wait = base × 2n + random(0, jitter)
変数デフォルト説明
base1000 ms初回リトライの基準待機時間
n0, 1, 2, ...リトライ回数（0始まり）
jitter1000 msランダム幅 0〜jitter ms
maxRetries4 回超えたら例外を投げる

04実装コード

fetchTweet — TypeScript / Node.js

TypeScript 
const OEMBED_ENDPOINT = 'https://publish.twitter.com/oembed';

interface OEmbedResponse {
  url:           string;
  author_name:   string;
  author_url:    string;
  html:          string;
  type:          string;
  cache_age:     string;
  provider_name: string;
  provider_url:  string;
}

interface RetryOptions {
  maxRetries?: number;  // default: 4
  baseMs?:     number;  // default: 1000
  jitterMs?:   number;  // default: 1000
}

/** 指数関数的リトライ付きツイート取得 */
async function fetchTweet(
  tweetUrl: string,
  opts: RetryOptions = {}
): Promise<OEmbedResponse> {
  const { maxRetries = 4, baseMs = 1000, jitterMs = 1000 } = opts;
  const apiUrl =
    `${OEMBED_ENDPOINT}?url=${encodeURIComponent(tweetUrl)}&omit_script=true`;

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    const res = await fetch(apiUrl);

    // ✅ 成功
    if (res.ok) return res.json() as Promise<OEmbedResponse>;

    // ❌ リトライ不要なエラー（鍵垢・削除済み）
    if (res.status === 404) {
      throw new Error(`Tweet not found or private: ${tweetUrl}`);
    }

    // ⚠️ リトライ上限に達した
    if (attempt === maxRetries) {
      throw new Error(
        `Max retries (${maxRetries}) exceeded. Last status: ${res.status}`
      );
    }

    // ⏳ 指数関数的バックオフ + ジッター
    const wait = baseMs * 2 ** attempt + Math.random() * jitterMs;
    console.warn(
      `[oEmbed] HTTP ${res.status} — retry ${attempt + 1}/${maxRetries}`,
      `in ${(wait / 1000).toFixed(1)}s`
    );
    await new Promise((r) => setTimeout(r, wait));
  }

  throw new Error('Unexpected error in fetchTweet');
}

本文テキストの抽出

html フィールドには埋め込み用HTMLが入っているため、本文テキストのみが必要な場合はパースが必要。

TypeScript 
import { JSDOM } from 'jsdom';  // npm install jsdom @types/jsdom

function extractText(html: string): string {
  const dom = new JSDOM(html);
  const p = dom.window.document.querySelector('blockquote p');
  return p?.textContent?.trim() ?? '';
}

// 使用例
const tweet = await fetchTweet('https://twitter.com/i/status/1234567890');
const text  = extractText(tweet.html);

console.log('本文:',   text);
console.log('投稿者:', tweet.author_name);

05キャッシュ戦略

レスポンスの cache_age フィールド（秒単位）をTTLとしてキャッシュすることで、不要なリクエストを削減できる。

TypeScript 
const cache = new Map<string, { data: OEmbedResponse; expiresAt: number }>();

async function fetchTweetCached(tweetUrl: string): Promise<OEmbedResponse> {
  const now = Date.now();
  const hit = cache.get(tweetUrl);

  if (hit && hit.expiresAt > now) {
    return hit.data;  // キャッシュヒット
  }

  const data = await fetchTweet(tweetUrl);
  const ttl  = parseInt(data.cache_age, 10) * 1000;  // ms に変換
  cache.set(tweetUrl, { data, expiresAt: now + ttl });
  return data;
}
⚠️

上記はオンメモリキャッシュのため、プロセス再起動でリセットされる。永続化が必要な場合は Redis や SQLite と組み合わせること。

06注意事項

  • 鍵垢・削除済みツイートは 404 になり、内容は取得不可能。リトライしても解決しない。
  • いいね数・RT数などのエンゲージメント指標は取得できない。メディア（画像・動画）の直URLも返却されない。
  • メディアが必要な場合は X API v2（Basic プラン以上・$100/月〜）を使用する。
  • Xは予告なく oEmbed の仕様やレスポンス形式を変更することがある。本番利用時はレスポンスの型バリデーションを推奨。
  • ブラウザから直接呼び出す場合は CORS の制限に注意。サーバー経由での呼び出しを推奨。
ℹ️

1日あたり数件程度の取得であれば、デフォルト設定のリトライで十分に安定稼働する。大量取得が必要になった場合は X API v2 への移行を検討すること。
